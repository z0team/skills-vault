# Agent Ingest Prompt

Промпт нижче признач для будь-якого coding-агента з доступом до bash/git
(Claude Code, Cursor Agent, тощо), запущеного **всередині директорії
`skills-vault`** (тобто `pwd` == корінь цього репо).

Встав посилання(-я) замість `<URL>` і віддай агенту. Можна декілька
посилань одразу, по одному в рядку.

---

## ПРОМПТ (копіюй усе нижче в чат агента)

```
Ти працюєш в репозиторії skills-vault — особистому сховищі AI-агентських
skill-паків. Структура репо описана в README.md, прочитай його перед
початком, якщо ще не читав.

ЗАВДАННЯ: інжестнути новий(і) skill-пак(и) за посиланням(-ями):

<URL>

Для КОЖНОГО посилання виконай послідовно:

1. ЗАПУСТИ ./scripts/ingest.sh <URL>
   Це склонує репо, знайде SKILL.md (або .mdc файли як fallback) і
   скопіює нормалізований пак у packs/<author>/<pack-name>/skills/.

2. ПЕРЕВІР якість результату для кожного доданого скіла:
   - Відкрий кожен packs/.../SKILL.md
   - Переконайся, що frontmatter має мінімум `name` (kebab-case, збігається
     з назвою директорії) і `description` (третя особа, формула: що робить
     + коли використовувати + тригер-фрази — НЕ "Helps with X", а конкретно)
   - Якщо description слабкий/відсутній (типове при конвертації з .mdc) —
     перепиши його сам, прочитавши тіло скіла, за тим самим стандартом, що
     використовує Claude Code (дивись приклади в /mnt/skills/public/*/SKILL.md
     якщо вони доступні в цьому середовищі, або в README.md цього репо)
   - Видали будь-який сміттєвий контент, що міг приїхати з джерела
     (badges, посилання на CI, незрозумілі релативні посилання, що вказують
     на файли поза packs/.../<skill-name>/)
   - Якщо в скілі є посилання на допоміжні файли (scripts/, templates/,
     references/) — перевір, що вони реально скопійовані поруч і шляхи
     в SKILL.md відносні (без абсолютних шляхів типу /home/... чи C:\...)

3. ПЕРЕВІР ліцензію джерела:
   - Глянь LICENSE/LICENSE.md/LICENSE.txt в склонованому репо (тимчасова
     директорія вже видалена ingest.sh, тому при потребі клонуй ще раз
     окремо або глянь pack.json — туди вже записана license_note)
   - Якщо ліцензія явно забороняє редистрибуцію (рідкість, але буває) —
     ЗУПИНИСЬ і повідом мене, не комить такий пак

4. ЯКЩО посилання вказує на МОНОРЕПО з купою непов'язаних скілів
   (наприклад awesome-list на 1000+ скілів) — НЕ тягни все підряд.
   Запитай мене, які саме скіли/категорії мене цікавлять, перш ніж
   інжестити сотні файлів сміття.

5. ОНОВИ registry.json:
   ./scripts/build_registry.py

6. ПОКАЖИ мені підсумок: скільки скілів додано, з якими назвами, і чи
   були скіли з проблемами (слабкий опис, відсутня ліцензія, конфлікт
   імен з уже існуючим скілом в репо).

7. ЯКЩО назва скіла вже існує в іншому паку (перевір по registry.json) —
   НЕ перезаписуй мовчки. Або перейменуй новий (додай суфікс автора), або
   запитай мене що робити.

8. Закомить зміни з осмисленим повідомленням у стилі:
   "feat(packs): add <pack-name> by <author> (<N> skills)"
   Якщо git remote вже налаштований і я не казав інше — НЕ роби push
   автоматично, тільки commit. Запитай чи пушити.

ВАЖЛИВО:
- Ніколи не редагуй вручну згенеровані файли поза packs/ — все, що в
  .cursor/, .github/instructions/, AGENTS.md тощо генерується через
  scripts/install.sh + scripts/convert.py, а не редагується напряму.
- SKILL.md — єдине джерело правди для кожного скіла.
- Якщо ingest.sh впав з помилкою (немає мережевого доступу, repo не
  знайдено, нестандартний формат) — не намагайся хакнути обхідний шлях
  мовчки, скажи мені що пішло не так і запропонуй варіанти.
```

## Якщо посилання — не GitHub (npm-пакет, Smithery, окремий .zip тощо)

`ingest.sh` поки підтримує тільки `github.com/owner/repo` URL (включно з
`/tree/branch/subpath`). Якщо джерело інше — скажи агенту прямим текстом,
наприклад:

```
Джерело https://www.npmjs.com/package/some-skills-pack — це npm-пакет, не
git repo. Знайди його GitHub-репозиторій (зазвичай в package.json -> 
repository, або на сторінці npm), і використай той URL для ingest.sh.
Якщо GitHub-репо немає — завантаж тарбол через npm pack <name>, розпакуй
у тимчасову директорію вручну, знайди SKILL.md/.mdc файли і скопіюй їх
в packs/<author>/<pack-name>/skills/ за тією ж структурою, що описана в
README.md, потім онови registry.json.
```

## Швидкий чек-лист "новий пак готовий"

- [ ] `packs/<author>/<pack-name>/pack.json` існує і містить `source_url`
- [ ] Кожен `skills/<name>/SKILL.md` має валідний `name` + `description`
- [ ] Жодних абсолютних шляхів і битих посилань всередині SKILL.md
- [ ] Ліцензія перевірена, занотована в `pack.json.license_note`
- [ ] `registry.json` перебудований (`./scripts/build_registry.py`)
- [ ] Конфліктів імен з існуючими скілами немає (або вирішені свідомо)
- [ ] Закомічено з людським commit message
